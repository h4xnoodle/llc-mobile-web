Native Apps vs. HTML5 Web Apps

Native Apps

Custom-built applications on a mobile platform

Code is written in platform-specific languages: Java [BlackBerry & Android], Objective-C [iOS]

Native Apps - Distribution Channels

Ability to download apps from specific distribution channels

HTML5 & Web Apps

Next version of web technologies:

  • HTML5
  • CSS3
  • Javascript

Combinations produce web apps

Designed to run on low-powered devices

No official distribution channel for all HTML5 apps

Mobile website -> App experience

HTML5 & Web Apps - Distribution Channels



No official appstore - need to do more promotion

Chrome Webstore, Opera's Widgets

Which one to choose?

Depends on your goals for the app


Native

  • Built specifically for the platform(s) (BB, iOS, Android)
  • Dependent on device version (eg. iOS 4 vs iOS 5)
  • Distribution channels (Appstore, Market)
  • Greater flexibility; usage of device features. eg Camera/mail

HTML5

  • Just a website - any platform that has a browser
  • Depends on device's browser - standards exist
  • No official distribution channel
  • Ability to update at any time - dist. channels require update process

HTML Basics

  • An HTML page is made up of elements commonly called tags
  • <p> ... </p> - paragraphs
  • <a href=""> ... </a> - links
  • <div> ... </div> - divs 'dividers of content'
  • and many more

These HTML tags define the structure of your page

<html>
  <head>
    <title>My Awesome Page</title>
  </head>
  <body>
    <p>First Paragraph</p>
  </body>
</html>

Note the nesting and closing of tags

Combine Structure and Style

How do we style our HTML elements with CSS?

<p class='my-css-class'>

We can style whole elements, or differentiate elements with classes

p.my-css-class {
  color:blue;
}

This is a block of CSS. The curly braces surround the properties

Example

<p>I'm just blue!</>
<p class='yellow'>I'm yellow :)</p>
p {
  color:blue;
}
p.yellow {
  color:blue;
  background-color:yellow;
}

Produces:

I'm just blue!

I'm yellow :)

In CSS, <p class='yellow'> became p.yellow

If just 'p' is specified, the styling applies to all 'p' tags

More than one class

You can specifiy multiple classes for an element

<p class='yellow purple'>I'm purple and yellow</p>

CSS:

p {
  color:blue;
}
p.yellow {
  background-color:yellow;
}
p.purple {
  color:purple;
}

I'm purple and yellow

Note how the color:blue; was overridden

When arranging content for small screens, we make things 'linear' down the screen.

More CSS - Floats

Floats allow you to take an HTML element out of its normal position, and slide it up to the left or right.

They allow you to easily create things like columns as in our desktop site

More CSS - Display

Some HTML elements naturally occur in the same line -- 'inline' -- with other elements. Inline elements usually do not support a height, width or padding as they sit inline anyways. Block elements like to take up as much horizontal space as possible.

Display: inline-block;

You can combine these two behaviours. The divs here would have a width and height (like our 'items')

Mobile Design - Linearization

Using our knowledge of display:, we can now very easily display our items one at a time down the screen.

Similarily, with our knowledge of float:, we can pop the sidebar and content back on top of each other.

Mobile browsers attemping to be intelligent - "zoom to fit screen"

The device is trying to be smart and showing you as much as possible... we don't want that though!

"Zoom to Fit" vs "Zoom to Actual Size"

Zoom to fit screen: Scale it! I want a birds-eye view of everything on the screen (50%)

Zoom to actual size: I want to see the actual size of my content (100%)

We need to specify a view behaviour for the browser. vs

The Viewport

My browser screen size ('visual viewport') is 320px

We need to set the viewable content behaviour to 'zoom-to-actual-size' (100%).

Restrict the amount of content (layout viewport) the browser will scale to, to the visual viewport

Touch Area - Fat Finger Problem

The touch area was too small

vs.

The mouse pointer is accurate to 1 pixel.

Our fingers need about 44px x 44px to be accurate.

We want to make sure all things we can interact with are at least this larger - or close to our finger size.

Even More CSS - Margins and Padding

Margins and Padding are used to create space between elements and space within elements.

Below is what is called the 'box model'. A box uses margins to separate itself from other elements, and uses padding to space its internal elements away from the edges

Super important for later

When the width of the box is set to, say, 200px, any padding added will increase the size of this box.

So if we say padding:10px; the box is really 220px wide now. We need to remember this for when we depend on widths for alignment.

Excruciating detail: http://www.w3.org/TR/CSS2/box.html

Exercise #10 - Enlarging the touch areas
